1.ArrayCaseStudy
โปรแกรมนี้ใช้ Array ในการเก็บคะแนนสอบของนักศึกษาจำนวน 10 คน เพื่อนำมาประมวลผลหาข้อมูลทางสถิติเบื้องต้น ได้แก่ ผลรวม, ค่าเฉลี่ย, คะแนนสูงสุด-ต่ำสุด, จำนวนคนที่สอบผ่าน (คะแนนตั้งแต่ 7 ขึ้นไป) รวมถึงระบุตัวนักศึกษาที่ต้องได้รับการทบทวนเนื้อหาเพิ่มเติม (คะแนนต่ำกว่า 5)

ผลลัพธ์จาก ArrayCaseStudy:
```text
คะแนนรวมของนักศึกษาทั้งหมด: 62
คะแนนเฉลี่ย: 6.2
คะแนนสูงสุด: 10
คะแนนต่ำสุด: 2
จำนวนนักศึกษาที่ได้คะแนนตั้งแต่ 7 คะแนนขึ้นไป: 5
รายชื่อนักศึกษาที่ควรได้รับการทบทวนเพิ่มเติม (คะแนนต่ำกว่า 5):
นักศึกษาคนที่ 3 ได้คะแนน 4
นักศึกษาคนที่ 8 ได้คะแนน 3
นักศึกษาคนที่ 10 ได้คะแนน 2
```

2.StackCaseStudy
โปรแกรมจำลองประวัติการทำงานและระบบ Undo โดยใช้ Stack ซึ่งโครงสร้างข้อมูลประเภทนี้ทำงานตามหลักการ LIFO (Last-In, First-Out) หรือ "เข้าหลังสุด ออกก่อนสุด" ทำให้เวลาเราสั่ง Undo ระบบจะดึงคำสั่งที่ผู้ใช้เพิ่งทำล่าสุดออกมาลบหรือย้อนกลับก่อนเสมอ

```text
ผลลัพธ์จาก StackCaseStudy:
คำสั่งทั้งหมดใน Stack (ล่างขึ้นบน):
 Type Data
 Type Structure
 Delete Structure
 Type Algorithm
 Type Java

กำลัง Undo 2 ครั้ง...
Undo: Type Java
Undo: Type Algorithm

สถานะของ Stack หลังจาก Undo:
 Type Data
 Type Structure
 Delete Structure

Stack ใช้หลักการ LIFO โดย:
คำสั่งที่เพิ่มเข้าไปล่าสุดจะถูก Undo ก่อนเสมอ
เหมือนกับการกด Ctrl+Z ที่ยกเลิกการกระทำล่าสุดก่อน
```

3.QueueCaseStudy
โปรแกรมจำลองระบบรับบัตรคิวและการเรียกคิวผู้ป่วยโดยใช้ Queue ซึ่งทำงานตามหลักการ FIFO (First-In, First-Out) หรือ "มาก่อน ได้ก่อน" ทำให้ผู้ป่วยที่มาลงทะเบียนและเข้าคิวก่อน จะได้รับการเรียกชื่อเพื่อรับบริการก่อนตามลำดับเวลา เพื่อความยุติธรรม

ผลลัพธ์จาก QueueCaseStudy:
```text
คิวเริ่มต้น: [P001, P002, P003, P004, P005]

เรียกผู้ป่วยเข้ารับบริการ:
เรียก: P001 เข้ารับบริการแล้ว
เรียก: P002 เข้ารับบริการแล้ว

เพิ่มผู้ป่วยใหม่ P006 และ P007 เข้าคิวแล้ว

ผู้ป่วยคนถัดไป: P003
จำนวนผู้ป่วยที่ยังรออยู่: 5 คน

สถานะของ Queue หลังจากดำเนินการทั้งหมด: [P003, P004, P005, P006, P007]

Queue ใช้หลักการ FIFO โดย:
ผู้ป่วยที่มาลงทะเบียนก่อนจะได้รับบริการก่อนเสมอ
เหมือนการต่อแถวในชีวิตจริงที่มาก่อนได้ก่อน
```
